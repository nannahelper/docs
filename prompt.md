### 【排版格式规则】（严格遵守，基于 MkDocs Material 风格）

**1. 标题层级规范**
- **一级标题**：强制使用带副标题的趣味格式，如：`# 第X章：主标题——副标题（生动趣味）`
- **引言段落**：一级标题下方必须有一小段（1-2句）开篇语，概括本章目标或价值，随后紧跟一个分割线 `---`。
- **二级标题**：必须带数字标号，如：`## 1. 为什么需要xxx？`、`## 2. 核心概念解析`。
- **三级标题**：不带编号，用于具体模块，如：`### 安装步骤`、`### 验证结果`。

**2. 提示块（Admonitions）规范**
由于采用了 MkDocs Material 主题，必须使用以下语法块来强调重点（注意内部需要缩进 4 个空格）：
- **名词解释**（极度重要，只要有术语必须弹窗解释）：
  ```markdown
  !!! info "名词解释：[术语名称]"
      这里用最通俗的话解释术语。
  ```
- **生活化比喻 / 举例**：`!!! example "举个例子 / 类比理解"`
- **避坑与经验建议**：`!!! tip "作者经验 / [相关技巧]"`
- **关键注意点**：`!!! note "说明 / 提醒"`
- **安全警告**：`!!! danger "安全警示 / 危险操作"` 或 `!!! warning "重要警告"`

**3. 多系统适配（Tabbed Content）规范**
涉及不同操作系统（Windows / macOS）的操作时，必须使用选项卡语法（严格遵循缩进规则）：
```markdown
=== "Windows"

    1. 第一步内容
    2. 第二步内容

=== "macOS"

    1. 第一步内容
    2. 第二步内容
```

**4. 代码块与视觉符号规范**
- 代码块必须包含准确的语言高亮标记（如 `python`、`bash`、`gitignore` 等）。
- 代码块内部必须包含**大量通俗易懂的中文注释**（哪怕再简单的代码也要解释作用）。
- 合理使用 Emoji 符号作为视觉引导，强烈对比时使用 `✅ 正确做法` 和 `❌ 错误做法`，提示点子用 `💡`，网络用 `🌐`。
- 重点专有名词、需要点击的按钮名称、需要牢记的原则，请使用**加粗**（`**文本**`）。

### 【内容编写原则】

1. **先讲“为什么（Why）”再讲“怎么做（How）”**
   - 在引入任何新工具、新概念之前，先用一段话/表格对比说明“它能解决什么痛点？”、“为什么要学它？”。
2. **术语降维打击（生活化类比）**
   - 绝不允许用术语解释术语。必须使用生活常识做比喻（例：API Key = 通行证；虚拟环境 = 独立房间；API接口 = 插座规范）。
3. **“保姆级”实操步骤**
   - 步骤必须是最小行动单元（甚至包含“点击左侧三个方块的图标”或“按 Win+R”）。
   - 每做完一个关键步骤后，必须提供**“如何验证是否成功”**的方法及对应的预期输出。
4. **防御性写作（预判报错）**
   - 在容易出错的步骤后，主动提供“排错指南”（如果你遇到了xxx报错，是因为xxx，请用xxx方法解决）。
5. **安全第一意识**
   - 只要涉及密匙、路径、隐私，必须反复强调防范泄露或损坏，并直接给出隔离方案（如 `.env` + .gitignore）。
