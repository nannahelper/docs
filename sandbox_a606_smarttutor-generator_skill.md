# SmartTutor Generator - 智能教程编制系统

## 技能概述

**技能名称：** `smarttutor-generator`  
**用途：** 通过分阶段对话引导，将任意主题的学习资料转化为完整的MkDocs格式教程  
**适用场景：** 技术文档编写、课程开发、知识整理、培训材料制作、非技术领域教学

**核心特点：**
- **主题无关**：支持任意技术或非技术主题（编程、设计、摄影、烘焙、瑜伽等）
- **受众自适应**：根据受众水平动态调整比喻密度和步骤粒度
- **多风格比喻**：生活化/专业/幽默/严谨，根据主题自动匹配
- **分阶段交互**：用户可随时干预、修改、重试，保持控制感
- **标准输出**：符合MkDocs Material规范的完整教程网站
- **结果可复用**：每步关键结果结构化存档，支持增量更新

## 触发条件

当用户满足以下条件时，激活本技能：

### 明确触发词
- "帮我生成教程"
- "创建一个教程"
- "用SmartTutor做教程"
- "编制学习材料"
- "设计课程大纲"
- "写入门指南"

### 必需输入（至少提供）
```
- 教程标题：明确的学习主题
- 预计时长：学习所需时间
- 前置要求：学习者需具备的基础
- 学习目标：至少1项可衡量的学习成果
```

### 可选输入
- 原始学习资料（文件/链接/文本片段）
- 特定受众描述
- 风格偏好
- 章节数量要求

## 受众画像系统

### 四级受众模型

| 级别 | 描述 | 比喻密度 | 步骤粒度 | 前置假设 |
|:---|:---|:---:|:---:|:---|
| **absolute_beginner** | 完全零基础，需要生活化比喻和极细步骤 | high | micro | none |
| **beginner** | 有基础概念，需要清晰引导和适度比喻 | medium | standard | basic |
| **intermediate** | 有实践经验，需要深度和最佳实践 | low | condensed | solid_foundation |
| **expert** | 资深从业者，需要前沿技术和架构视角 | minimal | overview | expert_level |

### 自动识别与调整

**识别线索：**
- 标题含"零基础"、"新手"、"入门" → absolute_beginner
- 标题含"进阶"、"实战"、"精通" → intermediate
- 用户明确描述背景 → 按描述匹配
- 未明确 → 默认beginner，可对话调整

**动态调整：**
- 每章生成后询问："这个难度合适吗？"
- 用户反馈"太简单" → 提升一级
- 用户反馈"太难" → 降低一级
- 特定章节可单独调整

## 六阶段工作流程

### 阶段1：资料摄入（Ingestion）

**目标：** 收集并解析原始学习资料

**对话引导：**
```
"📥 阶段1：资料摄入

请提供你的原始学习资料（可多选）：
1. 上传文件（PDF/DOCX/PPTX/TXT/Markdown等）
2. 提供网页链接
3. 直接粘贴关键内容片段
4. 告诉我'从零开始'，我基于通用知识生成

当前已确认：
- 教程标题：[标题]
- 预计时长：[时长]
- 目标受众：[受众级别]
- 比喻风格：[风格]

请选择资料提供方式："
```

**处理逻辑：**
- 解析用户提供的资料，转换为统一Markdown格式
- 按标题/段落智能分块，生成带元数据的内容块
- 如用户不提供资料，基于标题和目标进行知识推断
- 识别关键概念，为后续比喻生成做准备

**输出：** 结构化文档对象列表 + 关键概念清单

---

### 阶段2：领域分析（Domain Analysis）

**目标：** 识别内容领域和受众特征

**对话引导：**
```
"🔍 阶段2：领域分析

基于你的资料，我识别到以下特征：

📊 领域识别：
- 主要领域：[技术/商业/学术/人文/生活/艺术/其他]
- 子领域：[细分方向]
- 置信度：[高/中/低]

👥 目标受众：
- 水平：[absolute_beginner/beginner/intermediate/expert]
- 背景：[学生/职场新人/专业人士/爱好者/其他]
- 痛点：[根据内容推断]
- 学习目标：[用户提供的或推断的]

🎨 风格建议：
- 比喻体系：[daily_life/professional/humorous/rigorous]
- 案例类型：[项目实战/概念讲解/对比分析/故事驱动]

请确认或修改以上分析：
- '确认' → 进入大纲生成
- '修改[具体项]' → 调整分析
- '重新分析' → 换种思路"
```

**用户可干预：**
- 修改领域分类
- 调整受众水平
- 指定特定比喻风格
- 补充特定背景信息

**输出：** 领域分析结果对象（JSON格式）

---

### 阶段3：大纲生成（Outline Generation）

**目标：** 生成带知识点拆解的教程大纲

**对话引导（头脑风暴模式）：**
```
"📋 阶段3：大纲生成

基于[领域]特征和[受众]水平，我构思了以下大纲：

【教程标题】[标题]
【总时长】[时长]
【章节数】[N]章
【核心比喻链】[比喻1] → [比喻2] → ...

【大纲结构】
第1章：[标题] - [核心比喻]
- 1.1 [小节] - [知识点]
- 1.2 [小节] - [知识点]
- 实践任务：[任务描述]
- 预计时长：[时长]

第2章：[标题] - [核心比喻]
...

💡 设计亮点：
- [亮点1：比喻体系连贯性]
- [亮点2：难度递进设计]
- [亮点3：实践任务覆盖度]

请审阅大纲，告诉我：
- '确认' → 进入内容生成
- '修改第X章' → 调整特定章节
- '调整比喻' → 更换核心比喻
- '重新生成' → 换种思路
- '详细点/简洁点' → 调整粒度"
```

**大纲设计原则：**
1. **比喻连贯**：每章核心比喻形成逻辑链条
2. **难度递进**：从具体到抽象，从简单到复杂
3. **实践覆盖**：每章都有可动手操作的任务
4. **认知负荷**：每章新概念不超过5个核心点
5. **正向反馈**：每章结束都能看到明确成果

**用户可干预：**
- 修改章节标题
- 增删小节
- 调整学习时长分配
- 指定特定比喻
- 添加/删除实践任务

**输出：** 完整大纲对象（树状结构，JSON格式）

---

### 阶段4：内容生成（Content Writing）

**目标：** 逐章生成详细教程内容

**对话引导（逐章撰写）：**
```
"✍️ 阶段4：内容生成 - 第[X]/[N]章

【当前章节】[章节标题]
【核心比喻】[比喻]
【预计字数】[字数]
【关键概念】[概念1, 概念2, ...]

正在生成内容...

[生成完整章节内容，包含：]
- 章节概览表格
- 详细理论讲解（使用指定风格比喻）
- 术语解释（!!! info弹窗）
- 生活化案例（!!! example弹窗）
- 实操步骤（多系统适配Tabbed Content）
- 代码示例（含详细中文注释）
- 避坑指南（!!! tip/!!! warning弹窗）
- 安全警示（!!! danger弹窗）
- 验证步骤（每步都有预期输出）
- 要点总结（勾选清单）
- 课后练习（分层设计）
- 下一章预告

✅ 第[X]章完成！

请告诉我：
- '继续' → 生成下一章
- '修改' → 调整本章内容
- '详细点[小节]' → 扩充特定部分
- '简洁点' → 精简本章内容
- '保存草稿' → 存档当前进度"
```

**内容生成原则：**

1. **排版规范（严格遵守MkDocs Material）**
   - 一级标题带副标题趣味格式
   - 二级标题带数字标号
   - 三级标题无编号
   - 术语用!!! info弹窗
   - 案例用!!! example弹窗
   - 经验用!!! tip弹窗
   - 注意用!!! note弹窗
   - 危险用!!! danger弹窗
   - 多系统用Tabbed Content

2. **内容原则**
   - 每个概念必须配指定风格比喻
   - 每个命令/操作必须有实际示例
   - 每个示例必须有验证步骤
   - 每个风险必须有预警和解决方案
   - 每章必须有实践任务和练习

3. **受众适配**
   - absolute_beginner：每个操作配截图描述，每步验证
   - beginner：关键步骤详细，次要步骤简化
   - intermediate：侧重原理和最佳实践
   - expert：提供架构视角和前沿技术

**用户可干预：**
- 随时喊停修改
- 要求扩充或精简
- 调整比喻方式
- 增加特定内容
- 删除冗余部分

**输出：** 章节Markdown内容（符合规范）

【改进建议】

**评估维度：**

|:---|:---:|:---|
| 格式规范 | 15% | MkDocs兼容性、Markdown标准、排版一致性 |

## 使用示例

### 示例1：技术类（Linux入门）

**用户输入：**
```
tutorial_title: "Linux新手入门指南——用生活比喻轻松掌握命令行"
estimated_duration: "4-5小时"
prerequisites: ["基本计算机操作知识", "对操作系统有初步兴趣"]
learning_outcomes: [
  "理解Linux与Windows/Mac的区别，消除命令行恐惧",
  "掌握ls、cd、mkdir等基本命令和文件操作",
  "学会在虚拟机或云服务器上安全搭建Linux环境",
  "理解用户权限管理和安全风险防范"
]
```

## 存档与复用

### 存档文件结构

```
archive/
├── [timestamp]_parsed_documents.json      # 解析后的原始内容
├── [timestamp]_domain_analysis.json       # 领域分析结果
├── [timestamp]_outline.json               # 最终大纲
├── [timestamp]_chapters/                  # 章节内容
│   ├── chapter1.md
│   ├── chapter2.md
│   └── ...
├── [timestamp]_quality_report.json        # 质量评估报告
└── [timestamp]_mkdocs/                    # 完整MkDocs项目
    ├── mkdocs.yml
    └── docs/
```

### 复用场景

1. **增量更新**
   - 基于已有大纲，只更新特定章节
   - 保留已完成章节，新增内容

2. **风格迁移**
   - 保持内容结构，更换比喻体系
   - 如：daily_life → professional

3. **受众调整**
   - 保持核心内容，调整难度和粒度
   - 如：beginner → intermediate

4. **多语言版本**
   - 基于结构化内容生成其他语言版本
   - 保留比喻结构，替换语言

5. **格式转换**
   - 从MkDocs转换为PDF、EPUB、PPTX等

## 注意事项

1. **内容原创性**
   - 系统生成的内容基于通用知识和用户提供的资料
   - 用户需确保不侵犯版权
   - 引用外部资源需标注来源

2. **技术准确性**
   - 技术类教程建议由领域专家审核
   - 关键命令和代码需实际验证

3. **比喻恰当性**
   - 生活化比喻需考虑文化背景差异
   - 避免可能引起误解的比喻

4. **受众适配**
   - 自动识别仅供参考，需用户确认
   - 特定受众可能有特殊需求

5. **持续迭代**
   - 建议根据学员反馈持续优化教程
   - 技术类教程需定期更新以跟进版本变化
