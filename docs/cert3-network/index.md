# 计算机三级 · 网络技术

> **从"两台电脑怎么通信"到"全球互联网如何运转"** —— 系统掌握计算机网络核心原理，从容应对三级考试。

---

## 教程简介

计算机网络是计算机三级考试中最热门的选考科目之一。本教程以全国计算机等级考试三级网络技术考试大纲为依据，用生活化比喻拆解抽象的网络概念，帮助你从零开始系统掌握网络体系结构、协议原理和实用技术。

!!! info "为什么选择网络技术？"

    -  **实用性强** ：无论是 Web 开发、运维部署还是云计算，网络知识都是底层基础
    -  **考试友好** ：知识点体系化、逻辑清晰，适合系统学习
    -  **职业加分** ：网络工程师、安全工程师等岗位的必备技能

| 项目 | 内容 |
|:---|:---|
| **学习目标** | 掌握网络体系结构、核心协议原理，具备三级考试应试能力 |
| **预计时长** | 10~12 小时 |
| **前置要求** | 基本的计算机操作知识 |
| **考试大纲** | 网络体系结构、IP 地址与子网划分、路由协议、传输层、应用层、网络安全 |
| **实践任务** | 使用 Wireshark 抓包分析、配置简单网络拓扑 |

## 核心比喻链

本教程使用 **"快递物流体系"** 作为贯穿全篇的核心比喻：

| 网络概念 | 物流比喻 |
|:---|:---|
| 分层模型 | 物流公司的部门分工（收件部→分拣部→运输部→派送部） |
| IP 地址 | 收件地址（省市区街道门牌号） |
| 子网掩码 | 邮政编码（快速定位到哪个片区） |
| 路由器 | 中转站（根据地址决定下一站） |
| TCP | 挂号信（确认收件、按序送达、丢件重发） |
| UDP | 普通信件（尽力送达、不保证到达） |
| 端口号 | 收件人姓名（同一地址的不同人） |
| DNS | 邮编查询系统（名字→地址的转换） |
