# 第 3 章：发现与选择技能

> **打开技能世界的地图** —— 学会在技能生态中高效搜索、评估和选择最适合你的技能。

---

## 3.1 为什么需要"发现技能"？

AI 技能生态正在快速增长——每天都有新的技能被创建和分享。面对海量选择，"找到合适的技能"本身就是一项重要能力。盲目安装技能可能导致：

- 安装了功能重叠的技能，浪费 AI 助手的上下文窗口
- 使用了质量不高的技能，产出不尽如人意
- 错过了更适合你需求的替代方案

本章将教你一套系统的技能发现和评估方法。

---

## 3.2 技能搜索渠道

### 官方技能市场

| 平台 | 搜索方式 | 特点 |
|:---|:---|:---|
| **Claude Code** | `npx skills search <关键词>` | 命令行搜索，直接安装 |
| **skills.sh** | 网页浏览排行榜 | 能看到安装量、评分、分类 |
| **GitHub** | 搜索 `skills topic:claude-code` | 覆盖面最广，能看到源码和更新频率 |

### 命令行搜索（Claude Code）

```bash
# 搜索 PDF 相关技能
npx skills search pdf

# 搜索前端设计相关技能
npx skills search frontend design

# 搜索教程生成相关技能
npx skills search tutorial
```

### GitHub 搜索技巧

在 GitHub 搜索框中，使用以下关键词组合可以高效定位技能：

```
# 通用搜索
skills claude-code
skills cursor
skills trae-ide

# 按主题搜索
skills topic:claude-code
skills topic:ai-coding-assistant

# 按语言/框架搜索
claude-code skills python
claude-code skills react
```

## 3.3 技能评估清单

找到候选技能后，用以下清单进行评估：

### 质量评估

| 维度 | 检查项 | 好 | 差 |
|:---|:---|:---|:---|
| **活跃度** | 最后更新时间 | 近 3 个月内 | 超过 1 年未更新 |
| **社区认可** | Stars/安装量 | 有显著使用量 | 几乎无人使用 |
| **文档质量** | README 完整度 | 有使用示例和说明 | 只有几行描述 |
| **代码规范** | 目录结构 | 清晰、有 SKILL.md | 混乱、缺少关键文件 |
| **依赖情况** | 外部依赖 | 依赖少或声明清晰 | 大量未声明的依赖 |

### 匹配度评估

| 维度 | 检查项 |
|:---|:---|
| **功能匹配** | 技能的功能是否覆盖你的核心需求？ |
| **平台兼容** | 技能是否支持你使用的 AI 助手？ |
| **复杂度** | 技能的使用难度是否符合你的技术水平？ |
| **可定制性** | 技能是否支持自定义配置？ |

## 3.4 实战：搜索与评估流程

### 场景：我需要一个能处理 PDF 的技能

**第 1 步：在多个渠道搜索**

```bash
# Claude Code 命令行
npx skills search pdf
```
同时在 [skills.sh](https://skills.sh) 和 GitHub 上交叉搜索，对比结果。

**第 2 步：筛选候选**

从搜索结果中选出 2-3 个候选技能，关注它们的描述、安装量和最后更新时间。

**第 3 步：查看详情**

```bash
# 查看技能详细信息
npx skills view <技能名>
```

或直接访问 GitHub 仓库，阅读 README 和 `SKILL.md`。

**第 4 步：快速测试**

安装最有潜力的技能，用一个简单任务快速验证：

```
请用这个技能帮我提取 test.pdf 中的文字内容
```

**第 5 步：做出选择**

根据测试结果和评估清单，选择最适合的技能。如果都不满意，可以考虑自行创建（见第 4 章）。

## 3.6 避免常见的选技能误区

### 误区一：越多越好

安装过多技能会增大 AI 助手的上下文压力，反而降低响应质量。**按需安装，定期清理**。

### 误区二：只看安装量

安装量高不代表最适合你。一个"小而美"的冷门技能可能比一个"大而全"的热门技能更精准地满足你的需求。

### 误区三：忽视维护状态

技能依赖 AI 助手的接口和行为，而 AI 助手的更新可能使老旧技能失效。选择活跃维护的技能能减少这种风险。

### 误区四：不读 SKILL.md

`SKILL.md` 是技能的"说明书"。不阅读就直接使用，可能会错过重要功能或触发方式。

## 3.7 本章小结

- 在多个渠道交叉搜索，能获得更全面的技能列表
- 用"质量 + 匹配度"双维度清单评估候选技能
- 安装前先快速测试，确认技能是否满足需求
- 按需安装、定期清理，避免技能冗余
- 阅读 `SKILL.md` 是理解技能的最佳方式
