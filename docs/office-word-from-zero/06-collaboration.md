## 6.1 修订模式

### 修订模式的工作原理

**修订模式** 像一个"透明的修改记录仪"——它记录每一次增删改，但不直接修改原文。文档所有者可以逐条接受或拒绝修改。

| 操作 | 修订模式下的显示 |
|:---|:---|
| **插入文字** | 红色下划线 |
| **删除文字** | 红色删除线（或在气球中显示） |
| **格式修改** | 在右侧气球中说明 |

!!! info "开启修订模式"
    审阅 → 修订 → 点击"修订"按钮（或按 `Ctrl+Shift+E`）。开启后，按钮高亮显示，所有修改都会被追踪。

### 修订的显示方式

| 显示方式 | 说明 | 适用场景 |
|:---|:---|:---|
| **所有标记** | 显示所有修订痕迹 | 逐条审阅修改 |
| **简单标记** | 仅用左侧竖线标记修改位置 | 快速浏览哪些地方被改了 |
| **无标记** | 显示接受所有修订后的效果 | 预览最终效果 |
| **原始状态** | 显示修订前的原始文档 | 对比修改前后 |

### 接受和拒绝修订

| 操作 | 方法 |
|:---|:---|
| 逐条处理 | 右键修订标记 → 接受/拒绝 |
| 批量接受 | 审阅 → 接受 → 接受所有修订 |
| 批量拒绝 | 审阅 → 拒绝 → 拒绝所有修订 |
| 仅接受格式修订 | 审阅 → 接受 → 接受所有格式修订 |

!!! tip "审阅技巧"
    使用"上一条"和"下一条"按钮（审阅选项卡）逐条审阅，配合"简单标记"视图快速定位修改位置。

## 6.2 批注系统

### 批注 vs 修订

| 对比维度 | 修订 | 批注 |
|:---|:---|:---|
| 目的 | 直接修改文档内容 | 提出问题或建议 |
| 操作 | 增删改文字/格式 | 在文档旁边添加评论 |
| 处理方式 | 接受或拒绝 | 回复讨论后标记为已解决 |
| 对文档的影响 | 可能改变文档内容 | 不改变文档内容 |

### 批注的使用技巧

!!! example "实战：高效的批注工作流"
    1. 选中需要讨论的文字
    2. 审阅 → 新建批注（或 `Ctrl+Alt+M`）
    3. 输入你的问题或建议
    4. 对方可以 **回复批注** 进行讨论
    5. 问题解决后，右键批注 → 解决批注（批注变灰但不删除）

!!! tip "@提及"
    在批注中使用 `@用户名` 可以通知特定人员。这在 OneDrive/SharePoint 协作时特别有用。

## 6.3 版本比较与合并

### 比较文档

当你收到一份修改后的文档（对方没有使用修订模式），可以用"比较"功能找出所有差异。

!!! example "实战：比较两个版本的文档"
    1. 审阅 → 比较 → 比较
    2. 原文档：选择原始版本
    3. 修订的文档：选择修改后的版本
    4. 点击确定
    5. Word 生成一份显示所有差异的新文档

### 合并文档

当多人分别修改了同一文档的不同副本，可以用"合并"功能将修改汇总到一起。

| 操作 | 路径 |
|:---|:---|
| 比较两个文档 | 审阅 → 比较 → 比较 |
| 合并多个作者的修订 | 审阅 → 比较 → 合并 |

## 6.4 保护文档

### 限制编辑

| 保护类型 | 说明 | 适用场景 |
|:---|:---|:---|
| **修订模式强制** | 所有修改必须通过修订模式 | 需要审批的文档 |
| **批注模式** | 只能添加批注，不能直接修改 | 征求意见阶段 |
| **填写窗体** | 只能在内容控件中填写 | 表单/模板 |
| **只读** | 完全禁止编辑 | 最终发布的文档 |

!!! example "实战：设置文档保护"
    1. 审阅 → 限制编辑
    2. 勾选"仅允许在文档中进行此类型的编辑"
    3. 选择限制类型（如"修订"）
    4. 点击"是，启动强制保护"
    5. 设置密码（可选）

!!! warning "密码保护"
    Word 的编辑限制密码 **不是加密** ，有工具可以绕过。如需真正的文档安全，使用"文件 → 信息 → 保护文档 → 用密码加密"。

## 6.5 实时协作（Word Online / 桌面版）

### 协作模式对比

| 功能 | 桌面版 | Word Online |
|:---|:---|:---|
| 实时共同编辑 | ✅（需 OneDrive） | ✅ |
| 看到他人光标 | ✅ | ✅ |
| 修订模式 | ✅ | ✅ |
| 批注 | ✅ | ✅ |
| 版本历史 | ✅ | ✅ |
| 所有高级功能 | ✅ | ❌（仅基础功能） |

!!! tip "协作最佳实践"
    - 协作前约定使用修订模式还是直接编辑
    - 大段落重写前先在批注中说明意图
    - 定期检查版本历史，确保没有意外覆盖
    - 使用 OneDrive 或 SharePoint 存储文档以启用实时协作
