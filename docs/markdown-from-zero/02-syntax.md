# 第 2 章：核心语法

> **10 分钟掌握 Markdown 核心语法** —— 标题、段落、列表、链接、图片、代码，六大要素一网打尽。

---

## 2.1 标题

使用 `#` 号标记标题层级，`#` 的数量代表标题级别（1~6 级）。

```markdown
# 一级标题（最大）
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题（最小）
```

**渲染效果：** 一级标题字号最大、最醒目，通常用于文章标题；二级标题用于大章节；三级标题用于小节。六级标题的字号比正文还小。

!!! tip "标题规范"

    - `#` 后面要加一个空格再写文字
    - 一级标题通常一个文档只用一次（作为文档标题）
    - 建议使用二级和三级标题组织内容结构

## 2.2 段落与换行

```markdown
这是第一段文字。Markdown 中直接换行不会产生新段落，
仍然属于同一段。

这是第二段文字。空一行才会产生新的段落。

这是第一行。  
在行尾加两个空格再换行，  
可以实现段内换行（不产生段落间距）。
```

**渲染效果：** 段落之间有明显间距；段内换行（行尾两个空格）则紧密排列，没有额外间距。

## 2.3 文字格式

```markdown
**这是粗体文字**
*这是斜体文字*
***这是粗斜体文字***
~~这是删除线文字~~
`这是行内代码`
```

**渲染效果：** 粗体文字加粗显示，用于强调关键词；斜体文字倾斜，用于术语或书名；粗斜体两者叠加；删除线中间有一条横线；行内代码使用等宽字体和灰色背景。

## 2.4 列表

### 无序列表

```markdown
- 项目一
- 项目二
  - 子项目 2.1（缩进两个空格）
  - 子项目 2.2
- 项目三
```

**渲染效果：** 每项前面显示一个圆点（●），子项目缩进后使用空心圆（○）。

### 有序列表

```markdown
1. 第一步
2. 第二步
   1. 子步骤 2.1（缩进三个空格）
   2. 子步骤 2.2
3. 第三步
```

**渲染效果：** 每项前面显示递增的数字序号，子步骤使用小写字母或罗马数字。

## 2.5 链接

```markdown
[链接文字](https://example.com)

[带标题的链接](https://example.com "鼠标悬停时显示的文字")

<https://example.com>  <!-- 自动链接，直接写 URL -->
```

**渲染效果：** 链接文字显示为蓝色并带下划线，点击跳转到目标网址。带标题的链接在鼠标悬停时会显示提示文字。

## 2.6 图片

```markdown
![替代文字](image.png)

![带标题的图片](image.png "图片标题")

[![点击图片跳转](image.png)](https://example.com)  <!-- 图片链接 -->
```

**渲染效果：** 图片直接嵌入显示在文档中。如果图片加载失败，则显示替代文字。图片链接点击后会跳转到目标网址。

!!! warning "图片路径"

    推荐使用相对路径引用图片，如 `images/screenshot.png`。这样项目移动位置后图片仍然能正常显示。

## 2.7 代码

### 行内代码

```markdown
使用 `print()` 函数输出内容。
```

**渲染效果：** `print()` 使用等宽字体和浅灰色背景，与正文明显区分。

### 代码块

使用三个反引号包裹，并指定语言以获得语法高亮：

````markdown
​```python
def greet(name):
    """向用户打招呼"""
    return f"你好，{name}！"

print(greet("世界"))
​```
````

**渲染效果：** 代码块有独立的深色或浅色背景区域，关键字、字符串、函数名等会显示不同颜色（语法高亮），右上角通常有复制按钮。

## 2.8 引用

```markdown
> 这是一段引用文字。
> 引用可以有多行。
>
> 空一行后继续引用。

> 嵌套引用：
>> 这是第二层引用。
>>> 这是第三层引用。
```

**渲染效果：** 引用文字左侧有一条竖线，文字颜色略浅于正文。嵌套引用有多层竖线，层次分明。

## 2.9 分隔线

```markdown
---

***

___
```

**渲染效果：** 三个或更多的 `-`、`*`、`_` 会渲染为一条横跨页面的水平分隔线。
